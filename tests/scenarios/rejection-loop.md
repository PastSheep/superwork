# 场景：驳回循环

## 初始状态
- 执行组完成，产出物在 outputs/<task-id>/execution/
- 验证组第一次审查

## 预期行为
1. 验证组审查 → 发现阻断问题 → 驳回
2. 主 agent 读取 review-feedback.json
3. 主 agent 通知执行组：「被驳回了，修改意见：未注册网络包处理器」
4. 执行组从 T2 快照恢复上下文，修复问题
5. 第 2 轮验证 → 通过或再次驳回
6. 第 3 轮后仍无共识 → 主 agent 自主判断阻断/非阻断

## 验证方式
人工：实际跑一个小任务，在验证组中故意驳回 2 次，观察流转和恢复
